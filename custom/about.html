About Fire Arrow Server

Fire Arrow Server is a unified FHIR server that integrates dynamic Role-Based Access Control (RBAC) capabilities directly into the HAPI FHIR JPA starter project. This eliminates the need for separate facade services, improving performance and simplifying deployment while providing sophisticated healthcare data access controls.

Built on the mature HAPI FHIR framework with Java 17 and Spring Boot 3.x, it provides enterprise-grade healthcare data infrastructure with extensive customization capabilities.

Core Capabilities

Enterprise Security

  • OAuth 2.0 token-based authentication
  • FHIR R4 compartment-based authorization
  • Multi-tenant architecture support
  • Fine-grained access control

High Performance

  • <100ms cached validation response
  • <500ms uncached validation response
  • 1000+ concurrent users per tenant
  • Horizontal scaling support

FHIR Compliance

  • Complete FHIR R4/R5 implementation
  • Advanced search capabilities
  • Clinical reasoning (CQL)
  • CDS Hooks integration

Cloud-Native

  • Docker and Kubernetes ready
  • CNPG PostgreSQL integration
  • FluxCD GitOps compatibility
  • Prometheus metrics support
